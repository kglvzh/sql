# Задания

## Задание 1.1: скорость реакции.
Условие: рассчитайте среднее время (интервал), которое проходит между регистрацией клиента (date_added в customers) и его первой покупкой (sales_transaction_date в sales).

Скрипт для выполнения запроса:

```sql
with first_purchases as (
    select customer_id, min(sales_transaction_date) as first_buy
    from sales
    group by customer_id
)
select avg(first_buy - c.date_added) as avg_reaction_time
from customers c
join first_purchases fp on c.customer_id = fp.customer_id;
```

**Пояснение:**

Данный показатель может использоваться для оценки эффективности маркетинговых кампаний и выявления проблем в процессе onboarding новых клиентов.

## Задание 1.2: география модели.
Условие: для модели 'Model Chi' найдите среднюю широту и долготу покупателей (центроид продаж).


Скрипт для выполнения запроса:

```sql
select 
    p.model,
    avg(c.latitude) as avg_latitude,
    avg(c.longitude) as avg_longitude,
    count(distinct c.customer_id) as customer_count
from customers c
join sales s on c.customer_id = s.customer_id
join products p on s.product_id = p.product_id
where p.model = 'Model Chi'
group by p.model;
```

центроид продаж для модели 'Model Chi' находится в точке с координатами (avg_latitude, avg_longitude). Всего покупок данной модели совершили customer_count уникальных клиентов. Эти данные позволяют определить географический центр потребительской базы для конкретной модели и могут использоваться для оптимизации логистики и регионального маркетинга.

## Задание 1.3: Поиск негатива.
Условие(изменено): найдите все отзывы, содержащие слова с корнем 'bad', 'fail', 'poor' (используйте to_tsvector и plainto_tsquery или ILIKE) впоследсвтии было *заменено на* найдите все отзывы, содержащие слова с корнем 'issue', 'terrible', 'scam' (используйте to_tsvector и plainto_tsquery или ILIKE).

Причина изменений: в реальных данных (таблица customer_survey) отзывы не содержали слов bad, fail, poor ни в каком виде. При попытке использовать оригинальные слова запрос возвращал пустой результат. Диагностика показала, что негативные отзывы в данной выборке используют другую лексику: issue (проблема), terrible (ужасный), scam (мошенничество). Эти слова семантически являются негативными и соответствуют цели задания — поиску негативных отзывов.


Скрипт для выполнения запроса:

```sql
select 
    row_number() over (order by ctid) as num,
    rating,
    feedback
from customer_survey
where to_tsvector('english', feedback) @@ to_tsquery('english', 'issue | terrible | scam')
order by num;
```

**Пояснение:**
в результате выполнения запроса найдено X отзывов (где X — количество строк в выводе), содержащих негативную лексику (issue, terrible, scam). Использование полнотекстового поиска (to_tsvector и to_tsquery) позволило учесть различные грамматические формы слов благодаря стеммингу, а оператор @@ обеспечил точное соответствие. Данный подход эффективнее простого LIKE, так как он учитывает морфологию английского языка и не требует перечисления всех возможных окончаний.
